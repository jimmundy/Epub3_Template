Preface

Writing an ebook by coding (and by avoiding the use of a third party conversion software) can be cumbersome if one is new to HTML, XML, MathML, etc. After patiently holding on to my do-it-yourself courage, I succeeded in writing six e-books by mere coding (in both ePub 2 and ePub 3 formats). Thereafter, I considered it was a good idea to write this template which will speed up the writing process of any author in re-flowable ePub 3 format.

The major objective is to distribute the template without DRM (Digital Right Management) so that interested writers can easily download and re-use the eBook. To assist the the author or writer to easily utilize the template, it is made up of the following 10 XHTML files (in addition to the three basic files— 'mymetype', 'toc.ncx', 'container.xml' and content.opf):

  1. Cover page— (front-cover.xhtml)
  2. Title page— (titlepage.xhtml)
  3. Content Summary— (content_summary.xhtml)
  4. Navigation— (toc.xhtml)
  5. Preface— (preface.xhtml)
  6. Imprint— (imprint.xhtml)
  7. Chapter I— (chapter1.xhtml)
  8. Chapter II— (chapter2.xhtml)
  9. Chapter III— (chapter3.xhtml)
  10. Chapter IV— (chapter4.xhtml)

After unzipping the ePub, you will obviously see these XHTML files. "Lorem ipsum" is used as text in most of the paragraphs which is meant to reflect the book as a template. The writer can duplicate any of the XHTML files, rename them for the sake of additional chapters, references, etc. The trick is that the opf, ncx and the toc files must be updated accordingly.

The key is simple (see the example below): After opening each XHTML file on a text editor, replace the headers and/or titles with your own title headings. Remove headings you do not require and add extra headings as desired. Perhaps, an author prefers to use the term Introduction instead of Preface. You may change this name in the preface.xhtml file while still retaining the file name as preface.xhtml file.

Another crucial step is that when you change a header, the ‘id’ must be changed as well. If you want the 'id' to be consistent with the header. This means you change the 'id' in both the content document and the navigation document (toc.xhtml)

Mr Jason Example

Mr. Jason has already altered the opf file by inserting the book title (Fitness Basics), the ISBN, the publishing date, the book description, etc. He is now focusing on the first paragraph of the first chapter of his book, 'Fitness Basics' written in Microsoft Word, which should contain a 15-second video at the end of the first paragraph. He does the following to the template:

  1. Changes the first chapter title and replaces the paragraph text (Lorem Ipsum…) with the corresponding paragraph text
  2. Changes the heading 'This is a video title' to 'Your First Exercise'
  3. Uploads the desired video file to the 'video' folder and use the actual video file name within the video tag elements
  4. Goes to the 'content.opf' text to change the video file name as well as the 'id'

Chapter I includes the video and audio tags. Chapter II includes a mathematical equation using MathML and Chapter III includes the ePub 3 versatile Figure element. Finally, Chapter IV is an empty chapter that can be used by the author. Chapters can be added or deleted provided changes are made in all the relevant files— the opf file, the toc file, e.t.c. The CSS (stylesheet) used here is by all means not the ultimate choice for every writer. It can be modified. The author can make any changes as desired.

The Caution Term

Subtiles termed Caution are added within two of the four chapters and can be easily deleted by the author when the ePub is cracked-open. It is meant to point out some extra specifics to assist the author. As an example, it is noted in the video embedding section (Chapter1) that two different taggings were used for this publication. - one ePub file with M4V video format channelled to Apple iBooks and another with an MP4 file commonly used on several e-readers and as a requirement by distributors and retailers. The later can be easily uploaded to channels like Kindle Direct Publishing (KDP). Audio must be encoded using M4A file format for a target towards Apple iBooks store while the popular MP3 suits many other e-readers.
